Program Basics
source file · main · statements · blocks
Your First C Program
#include <stdio.h> int main(void) { printf("Hello, AMIT!\n"); return 0; }

#include <stdio.h>

Includes declarations for standard input/output features such as printf. The Preprocessor section explains headers in depth later.

main(void)

The program begins at main. Writing void explicitly means the function takes no parameters.

Statements

Most C statements end with a semicolon, such as return 0;. A missing semicolon is a common early compiler error.

Blocks

Curly braces { } group statements into a block. Blocks define the body of functions, conditions, and loops.

Compile, Run, Read the Result
// Typical desktop GCC command gcc -std=c11 -Wall -Wextra hello.c -o hello // Run the generated program: ./hello on macOS/Linux, or hello.exe on Windows ./hello
Start by reading compiler messages from top to bottom. The first error is often the root cause; later errors may only be consequences of it.

Size Notes

Portable Thinking

C only guarantees minimum relationships such as sizeof(char) <= sizeof(short) <= sizeof(int) <= sizeof(long). Exact sizes depend on the platform.

Check at Runtime

Use sizeof(type) and limits from headers like <limits.h> and <float.h> when you need exact information.

What Modifiers Do

Size Modifiers

short usually reduces storage and range.
long increases range.
long long provides at least 64-bit integer storage.

Sign Modifiers

signed allows negative and positive values.
unsigned stores only non-negative values and increases positive range.

Scope Rules

Block Scope

Variables declared inside {} exist only within that block. Inner blocks shadow outer variables with the same name.

File Scope

Global variables declared outside all functions. Visible from declaration point to end of file. Marked static for internal linkage.

int g = 10; // File scope (global) void f() { int g = 20; // Shadows global g { int g = 30; // Shadows both printf("%d", g); // 30 } printf("%d", g); // 20 }

Casting & Branching
type conversion · if/else · switch
Type Casting
Syntax (type_name) expression
// Implicit conversions (automatic) int i = 3; float f = i; // int → float: implicit conversion double d = f; // float → double: implicit conversion // Integer promotion example: char is promoted to int in this expression char small = 10; int total = small + 1; // Explicit casting (manual) int a = 7, b = 2; float result = (float)a / b; // 3.5 (without cast: integer division gives 3) double pi = 3.14159; int truncated = (int)pi; // 3 (truncation, not rounding)

Usual Arithmetic Conversions

In mixed expressions, C applies integer promotions and usual arithmetic conversions. Floating types generally dominate integer types, but signed/unsigned integer combinations follow conversion-rank rules.

A cast can lose information or hide a mistake. Use an explicit cast only when you can explain why it is safe. Pointer conversions are introduced later in the pointer module.
Branching — if / else
if (condition) { // executed if condition is true (non-zero) } else if (another) { // else-if chain } else { // fallback } // Dangling else problem: if (a) if (b) x(); else y(); // else pairs with NEAREST if (b), NOT (a)! // Use braces to avoid ambiguity.
Branching — switch
switch (expression) { // Must be integer type case 1: printf("One\n"); break; // Without break, falls through! case 2: case 3: // Cases 2 & 3 share code printf("Two or Three\n"); break; default: printf("Other\n"); break; // Good practice, but optional }
switch only works with integral types (int, char, enum). Case labels must be constant expressions. Always handle default.

Loops
for · while · do-while · break · continue
for Loop
Syntax for (init; condition; update) { body }
// Standard for loop for (int i = 0; i < 10; i++) { printf("%d ", i); // 0 1 2 ... 9 } // Multiple expressions for (int i = 0, j = 9; i < j; i++, j--) { printf("i=%d j=%d\n", i, j); } // Infinite loop for (;;) { /* runs forever */ } // Loop with no body for (i = 0; str[i] != '\0'; i++); // Just count length // C99: declare variable in init for (int i = 0; ...); // i scoped to loop only
while Loop
// Entry-controlled: condition checked BEFORE body while (condition) { // body may execute 0 or more times } // Common pattern: read until sentinel int x; while (scanf("%d", &x) == 1) { printf("Read: %d\n", x); }
do-while Loop
// Exit-controlled: body runs AT LEAST ONCE int choice; do { printf("Enter 1-5: "); scanf("%d", &choice); } while (choice < 1 || choice > 5); // NOTE the semicolon!
break & continue
for (int i = 0; i < 100; i++) { if (i == 5) break; // Exit entire loop immediately if (i % 2 == 0) continue; // Skip to next iteration printf("%d ", i); // 1 3 (5 never reached) } // break only exits ONE level of nesting // For multi-level breaks, use flags or goto
goto
// Generally AVOID, but useful for nested loop exits: for (int i = 0; i < n; i++) for (int j = 0; j < m; j++) if (found) goto cleanup; cleanup: printf("Done\n");
Prefer for loops when the iteration count is known. Prefer while when it's driven by a condition. Use do-while when the body must run at least once.

1D Arrays

Declaration (Prototype)
Syntax return_type function_name(param_type param_name, ...);

Definition (Implementation)

Parameter Passing

Pass by Value

Default in C. Function gets a copy. Changes inside the function do NOT affect the original.

void try_change(int x) { x = 99; // No effect on caller }

Pass by Pointer

Simulates pass-by-reference. Pass the address so the function can modify the original.

void actually_change(int *x) { *x = 99; // Modifies caller's var }
Variable Arguments
#include <stdarg.h> int sum(int count, ...) { va_list args; va_start(args, count); int total = 0; for (int i = 0; i < count; i++) total += va_arg(args, int); va_end(args); return total; } sum(3, 10, 20, 30); // 60
Return Values

Recursion
base case · recursive case · examples · pitfalls
Anatomy of Recursion
return_type recursive(params) { if (base_condition) { return base_value; // BASE CASE (stops recursion) } // Recursive case: call itself with smaller input return recursive(smaller_params); }
Classic Examples
// Factorial: n! = n * (n-1)! long fact(int n) { if (n <= 1) return 1; return n * fact(n - 1); } // fact(5) → 5*4*3*2*1 = 120
// Fibonacci: F(n) = F(n-1) + F(n-2) int fib(int n) { if (n <= 1) return n; return fib(n-1) + fib(n-2); } // fib(6) → 8 // WARNING: O(2^n) time!
// Tower of Hanoi void hanoi(int n, char from, char to, char aux) { if (n == 1) { printf("%c → %c\n", from, to); return; } hanoi(n-1, from, aux, to); hanoi(1, from, to, aux); hanoi(n-1, aux, to, from); }
// Binary Search (recursive) int binary_search_recursive(int arr[], int lo, int hi, int target) { if (lo > hi) return -1; int mid = lo + (hi - lo) / 2; if (arr[mid] == target) return mid; if (arr[mid] > target) return binary_search_recursive(arr, lo, mid-1, target); return binary_search_recursive(arr, mid+1, hi, target); }
Recursion vs Iteration
Recursion
Elegant, matches mathematical definition
Uses call stack (O(n) space typically)
Can cause stack overflow
Often slower (function call overhead)
Easier to write for trees/graphs
VS
Iteration
More verbose but explicit
Constant space (O(1) typically)
No stack overflow risk
Generally faster
Better for simple linear processes
Every recursive function MUST have a base case that is guaranteed to be reached. Without it: infinite recursion → stack overflow → undefined behavior.
Tail Recursion
// Tail recursion: recursive call is the LAST operation int fact_tail(int n, int acc) { if (n <= 1) return acc; return fact_tail(n - 1, n * acc); // No computation after call } fact_tail(5, 1); // 120 // Compilers can optimize tail calls to reuse the stack frame // (tail call optimization — not guaranteed in C standard)

Structs
definition · initialization · nesting · pointers · typedef
Definition & Initialization
// Define a struct struct Point { double x; double y; }; // Create and initialize struct Point p1 = {3.0, 4.0}; // Positional init struct Point p2 = {.y = 5.0, .x = 2.0}; // Designated init (C99) struct Point p3; // Uninitialized struct Point p4 = {0}; // All members zeroed // Access members with dot operator p1.x = 10.0; printf("(%.1f, %.1f)\n", p1.x, p1.y);
typedef with Structs
typedef struct { char name[50]; int age; float gpa; } Student; // Now use without 'struct' keyword: Student s1 = {"Alice", 20, 3.8}; Student s2 = {.age = 22, .gpa = 3.5, .name = "Bob"};
Nested Structs
typedef struct { int day, month, year; } Date; typedef struct { char title[100]; char author[50]; Date published; // Nested struct float price; } Book; Book b = { .title = "The C Programming Language", .author = "K&R", .published = {.day=22, .month=2, .year=1978}, .price = 52.99 }; printf("%d", b.published.year); // 1978
Accessing Members
Student s = {"Charlie", 21, 3.9}; printf("%s is %d\n", s.name, s.age); s.age = 22; // Passing a struct by value copies the complete struct void print_student(Student student) { printf("%s, age %d\n", student.name, student.age); }
Use the dot operator . with a struct object. Later, the Advanced Pointers section introduces -> and passing a pointer to a struct when copying is undesirable.
Memory & Alignment
struct Example { char c; // 1 byte + 3 padding int i; // 4 bytes char d; // 1 byte + 3 padding }; // sizeof may be 12, not 6 (due to alignment) // Reorder for smaller size: struct Optimized { int i; // 4 bytes char c; // 1 byte char d; // 1 byte + 2 padding }; // sizeof may be 8 // Flexible array member (C99): struct Buffer { size_t len; char data[]; // Must be last member };
Structs are passed by value by default, which creates a full copy. Once pointers are introduced, you can pass a pointer to avoid that copy; use a const pointer when the function should only read the struct.

Enums & Unions
enumerations · union types · bit-fields
Enumerations
// Basic enum (values start at 0 by default) enum Day { MON, TUE, WED, THU, FRI, SAT, SUN }; // MON=0, TUE=1, ..., SUN=6 // Custom values enum Month { JAN = 1, FEB, MAR, APR, MAY, JUN, JUL, AUG, SEP, OCT, NOV, DEC }; // FEB=2, MAR=3, ..., DEC=12 // Non-sequential values enum ErrorCode { OK = 0, ERR_INVALID = -1, ERR_NOT_FOUND = -2, ERR_PERMISSION = -3 }; enum Day today = WED; printf("%d\n", today); // 2 // Use with switch (compiler can warn about missing cases) switch (today) { case MON: case TUE: case WED: case THU: case FRI: printf("Weekday\n"); break; case SAT: case SUN: printf("Weekend\n"); break; } // Typedef for cleaner syntax typedef enum { RED, GREEN, BLUE } Color; Color c = GREEN;
Unions
// Union: all members share the SAME memory // Size = size of LARGEST member union Data { int i; float f; char str[20]; }; union Data d; d.i = 42; printf("%d\n", d.i); // OK — last written was i // Do not treat this as numeric conversion:\n// interpreting d.f after writing d.i is non-portable and may be meaningless. // Typical portable pattern: tagged union (variant) typedef struct { enum { INT_VAL, FLOAT_VAL, STR_VAL } type; union { int ival; float fval; char sval[20]; } value; } Variant; Variant v; v.type = FLOAT_VAL; v.value.fval = 3.14f;
Bit-fields
// Specify exact bit width for each member struct Flags { unsigned int is_active : 1; // 1 bit unsigned int priority : 3; // 3 bits (0-7) unsigned int code : 4; // 4 bits (0-15) unsigned int : 0; // Force alignment to next boundary unsigned int id : 8; // 8 bits (0-255) }; struct Flags f = {1, 5, 10, 200}; printf("active=%d, pri=%d\n", f.is_active, f.priority);
Bit-fields are implementation-dependent: ordering, padding, and sign are not standardized. They're useful for hardware register mapping and saving memory, but avoid them in portable code.
Struct vs Union Quick Comparison
struct
Each member has own memory
Size = sum of members + padding
Can use all members at once
Most common composite type
VS
union
Members share same memory
Size is at least the largest member, plus possible padding
Track which member is currently active
Useful for tagged variants and compact storage

#define SQUARE(x) x * x is wrong because SQUARE(2+3) expands to 2+3 * 2+3. Even the parenthesized macro form can evaluate an argument more than once: never write MAX(i++, j++) or SQUARE(i++). ARRAY_COUNT(a) works only for a real array in the current scope, not for a pointer.

Pointer Fundamentals
addresses · dereferencing · pointer types · null
What Is a Pointer?

A pointer is a variable that stores the address of another object or function. The & operator gets an address, and the unary * operator dereferences a pointer to access the value at that address.

#include <stdio.h> int main(void) { int value = 42; int *ptr = &value; printf("value = %d\n", value); printf("address = %p\n", (void *)ptr); printf("through pointer = %d\n", *ptr); *ptr = 100; printf("new value = %d\n", value); return 0; }
Common Pointer Declarations
DeclarationMeaning
int *p;Pointer to int
char *text;Pointer to char; often points to the first character of a string
float *fp;Pointer to float
void *data;Generic pointer to an object; convert to the correct object-pointer type before dereferencing
int **pp;Pointer to a pointer to int
const int *p;Pointer to a const int; cannot modify *p through this pointer
int * const p = &x;Const pointer; its stored address cannot change after initialization
const int * const p = &x;Const pointer to a const int
Pointer to Pointer
int value = 7; int *ptr = &value; int **ptr_to_ptr = &ptr; **ptr_to_ptr = 21; printf("%d\n", value); // 21
NULL Pointer
#include <stddef.h> int *ptr = NULL; if (ptr != NULL) { printf("%d\n", *ptr); }
NULL is a null pointer constant. When assigned to a pointer, it creates a pointer value that does not point to an object or function. It is safe to compare or assign, but dereferencing it is undefined behavior.
Read complex declarations from the variable name outward. For example, int *p means “p is a pointer to int,” while int (*p)[5] means “p is a pointer to an array of 5 ints.”

Pointer Safety
null · wild · dangling · array safety
Unsafe Pointer States
StateWhat It MeansSafe Response
NULL pointerIntentionally points to no objectCheck before dereferencing; initialize optional pointers to NULL
Wild pointerAn uninitialized pointer with an indeterminate valueInitialize it immediately, usually to NULL or a valid address
Dangling pointerStill stores an old address after the pointed-to object lifetime has endedDo not use it; reset it to NULL after free when appropriate
Wild Pointer
int *wild; // *wild = 10; // Undefined behavior: wild is uninitialized wild = NULL; // Safe initial state
Dangling Pointer
int *numbers = malloc(4 * sizeof *numbers); if (numbers != NULL) { free(numbers); numbers = NULL; }
Pointers and Arrays
int values[] = { 10, 20, 30, 40 }; int *ptr = values; // points to values[0] printf("%d\n", *ptr); // 10 printf("%d\n", *(ptr + 2)); // 30 printf("%d\n", values[2]); // same value as *(values + 2) ptr++; printf("%d\n", *ptr); // 20
Pointer arithmetic scales by the pointed-to type. You may form a one-past-the-end pointer for iteration, but you must never dereference it. Relational comparisons such as < and > are meaningful only for pointers into the same array object (or one past it).
Lifetime Reminder

Automatic Objects

Local variables normally stop existing when their enclosing block or function ends. Returning their address creates a dangling pointer.

Dynamic Objects

Heap allocations remain valid until they are released with free. The next page covers allocation, resizing, ownership, and freeing in detail.

Returning Pointers Safely
int *bad_pointer(void) { int local = 10; return &local; // Wrong: local stops existing when the function returns } void set_value(int *out) { if (out != NULL) { *out = 10; } }

Memory Management
heap · malloc · calloc · realloc · free
Automatic Memory vs. Dynamic Memory
StorageExampleLifetimeWho Releases It?
Automatic / localint score = 0;Until its block or function endsCompiler/runtime automatically
Dynamic / heapint *scores = malloc(...);Until free is calledYour program
Static / globalstatic int count;Entire program executionRuntime at program exit
Use dynamic memory only when the required size is known at runtime, when data must outlive the current function, or when a large object should not live on the stack.
Allocating Memory: malloc and calloc

malloc

malloc(bytes) reserves a block of uninitialized memory. Read or write it only after you confirm that allocation succeeded.

calloc

calloc(count, size) reserves space for multiple elements and initializes all allocated bytes to zero.

#include <stdlib.h> size_t count = 5; int *scores = malloc(count * sizeof *scores); if (scores == NULL) { // Allocation failed: report, recover, or return an error. } else { scores[0] = 95; // malloc memory is ready after initialization } int *zeros = calloc(count, sizeof *zeros); if (zeros != NULL) { // The allocated bytes start as zero. }
Resizing Memory: realloc
size_t new_count = 10; int *resized = realloc(scores, new_count * sizeof *scores); if (resized != NULL) { scores = resized; // It may be the same address or a new address. } else { // The original scores allocation is still valid here. // Do not overwrite scores directly with realloc(...). }
Never write scores = realloc(scores, ...) unless you can safely lose the original pointer on failure. Store the result in a temporary pointer first. Avoid using realloc(ptr, 0) as a free shortcut; call free(ptr) explicitly when releasing memory.
Releasing Memory: free
free(scores); // Release a valid allocation once. scores = NULL; // Prevent accidental use through this pointer variable. free(scores); // Safe: free(NULL) does nothing. free(zeros); zeros = NULL;
Only call free for pointers returned by memory-allocation functions such as malloc, calloc, or realloc. Do not free a local array, a string literal, or an interior pointer such as scores + 1.
Common Memory-Management Bugs
BugExamplePrevention
Memory leakAllocation is no longer reachable and was never freedDefine who owns each allocation and free it once.
Double freeCalling free twice on the same allocationSet the owner pointer to NULL after freeing; avoid duplicate owners.
Use after freeDereferencing a pointer after freeDo not use the pointer again; clear it and clear aliases when possible.
Invalid freeFreeing a stack variable or an interior addressFree only the original allocation pointer.
Unchecked allocationUsing a NULL result as if it were validCheck the result before accessing the memory.
Size overflowcount * sizeof *scores wraps before allocationValidate large counts before multiplying.
Ownership Pattern
int *create_scores(size_t count) { int *result = calloc(count, sizeof *result); if (result == NULL) { return NULL; } return result; // Caller becomes responsible for free(result). } // int *scores = create_scores(30); // ... use scores ... // free(scores);

Advanced Pointers
structs · unions · arrays · functions · callbacks
Pointers to Structs
struct Sensor { int id; float temperature; }; struct Sensor room = { 1, 24.5f }; struct Sensor *sensor_ptr = &room; sensor_ptr->temperature = 25.0f; printf("Sensor %d: %.1f\n", sensor_ptr->id, sensor_ptr->temperature);
ptr->member is shorthand for (*ptr).member. Parentheses matter because the member-access operator . has higher precedence than unary *.
Pointer Inside a Struct: Linked Nodes
struct Node { int value; struct Node *next; }; struct Node first = { 10, NULL }; struct Node second = { 20, NULL }; first.next = &second; printf("%d\n", first.next->value); // 20
Pointers to Unions
union Measurement { int raw; float voltage; }; union Measurement reading; union Measurement *reading_ptr = &reading; reading_ptr->voltage = 3.3f; printf("%.1f\n", reading_ptr->voltage);
A union stores one member representation at a time. For portable code, track which member was most recently written and read that member.
Pointer to Array vs. Array of Pointers
DeclarationMeaning
int (*row_ptr)[4];Pointer to one array containing 4 int values
int *rows[4];Array containing 4 pointers to int
int row[4] = { 1, 2, 3, 4 }; int (*row_ptr)[4] = &row; printf("%d\n", (*row_ptr)[2]); // 3
Function Pointers
int add(int a, int b) { return a + b; } int subtract(int a, int b) { return a - b; } int (*operation)(int, int) = add; printf("%d\n", operation(7, 2)); // 9 operation = subtract; printf("%d\n", operation(7, 2)); // 5
Struct Holding Function Pointers: Callbacks
typedef int (*ReadFunction)(void); struct Device { const char *name; void (*start)(void); ReadFunction read_value; }; void start_sensor(void) { printf("Sensor started\n"); } int read_sensor(void) { return 42; } struct Device sensor = { "Temperature", start_sensor, read_sensor }; struct Device *device_ptr = &sensor; device_ptr->start(); printf("%s = %d\n", device_ptr->name, device_ptr->read_value());
Advanced Qualifier: restrict
// Promise: these pointer-based accesses do not overlap during this call void add_arrays( size_t count, int *restrict out, const int *restrict left, const int *restrict right ) { for (size_t i = 0; i < count; ++i) { out[i] = left[i] + right[i]; } }
restrict is a contract, not a cast. Do not use it unless you can guarantee the relevant accessed arrays do not overlap; breaking that promise can produce undefined behavior.
Pointer to a Function Pointer
int (*operation)(int, int) = add; int (**operation_slot)(int, int) = &operation; *operation_slot = subtract; printf("%d\n", operation(9, 4)); // 5
Function pointers are commonly used for callbacks, menus, drivers, event handlers, and hardware abstraction layers. Call a function through a pointer only when the pointed-to function has a compatible signature.

File Handling
open · read · write · position · close
The FILE Pointer
Header & Type #include <stdio.h>
FILE *file;
FILE * is a stream handle. It represents an open connection to a file, standard input/output, or another stream.
Opening a File — fopen
Syntax FILE *fopen(const char *filename, const char *mode);
FILE *file = fopen("students.txt", "r"); if (file == NULL) { perror("Could not open students.txt"); return 1; } // Use the file here... if (fclose(file) == EOF) { perror("Could not close file"); return 1; } file = NULL;
File Modes
ModePurposeFile Must Exist?Existing Contents
"r"Read onlyYesPreserved
"w"Write onlyNo — created if absentTruncated to empty
"a"Append onlyNo — created if absentPreserved; writes go to end
"r+"Read and writeYesPreserved
"w+"Read and writeNo — created if absentTruncated to empty
"a+"Read and appendNo — created if absentPreserved; writes go to end
"b"Binary modifierCombine with a modeExample: "rb", "wb", "ab+"
Use b for binary data. It matters on systems that translate text newlines, especially Windows. In an update stream such as "r+": after writing, call fflush or a positioning function before reading; after reading, call a positioning function before writing unless the read reached EOF.
Writing Text Files
FILE *out = fopen("report.txt", "w"); if (out == NULL) { perror("fopen"); return 1; } fprintf(out, "Name: %s\nScore: %d\n", name, score); fputs("Saved successfully\n", out); fputc('!', out); fclose(out);

Text Output Functions

fprintf writes formatted text.
fputs writes a string without adding a newline.
fputc writes one character.

Check the result of fclose after important output: buffered write errors may be reported only when the stream is closed.

Reading Text Files
char line[256]; FILE *in = fopen("report.txt", "r"); if (in == NULL) { perror("fopen"); return 1; } while (fgets(line, sizeof line, in) != NULL) { printf("%s", line); } if (ferror(in)) { perror("Read error"); } fclose(in);

Text Input Functions

fgets reads up to size - 1 characters and adds \0. It keeps the newline if one is read.

fscanf reads formatted input. Use width limits for strings, for example %49s.

fgetc returns an int so it can represent every unsigned character value and EOF.

Do not write while (!feof(file)). The EOF indicator is set only after a read attempts to go past the end. Loop using the actual read result, such as while (fgets(...)) or while ((ch = fgetc(file)) != EOF).
This section assumes hosted/desktop C with a file system. Bare-metal AVR programs usually use UART, EEPROM, flash, SD-card libraries, or host-side tools instead of standard disk files.

File Handling II
binary data · positioning · errors · file operations
Binary Files — fread & fwrite
Syntax size_t fread(void *ptr, size_t size, size_t count, FILE *stream);
size_t fwrite(const void *ptr, size_t size, size_t count, FILE *stream);
struct Student { char name[50]; int score; }; struct Student student = { "Mona", 95 }; FILE *bin = fopen("student.dat", "wb"); if (bin != NULL) { if (fwrite(&student, sizeof student, 1, bin) != 1) { perror("fwrite"); } fclose(bin); } // Reading: fread(&student, sizeof student, 1, bin)
fread and fwrite return the number of elements processed, not the number of bytes. Do not use raw struct dumps as a long-term portable file format: padding, byte order, pointer values, and type sizes can vary between builds and systems.
Moving the File Position
FILE *file = fopen("report.bin", "rb"); if (file == NULL) { perror("fopen"); return 1; } if (fseek(file, 0, SEEK_END) != 0) { perror("fseek"); } else { long bytes = ftell(file); if (bytes == -1L) { perror("ftell"); } } rewind(file); // Back to start; clears EOF/error flags fseek(file, 10, SEEK_SET); // 10 bytes from beginning fseek(file, -5, SEEK_END); // 5 bytes before end (binary files) fclose(file);

Position Constants

SEEK_SET — beginning
SEEK_CUR — current position
SEEK_END — end of file

Positioning Notes

For text streams, nonzero offsets with fseek are not fully portable. Use binary mode when you need byte-level positioning.

clearerr(file)Clears EOF and error flagsDoes not move the file position

Rename and Remove
if (rename("draft.txt", "final.txt") != 0) { perror("rename"); } if (remove("temporary.txt") != 0) { perror("remove"); }

Debugging & Logging
source locations · trace output · assertions · diagnostics
Debugging Mindset

Observe First

Make the failure repeatable, record inputs and environment details, then add the smallest useful amount of logging around the suspicious code path.

Log to stderr

Use stderr for diagnostics so normal program output on stdout can still be redirected or processed independently.

Built-In Source Information
NameWhat It Gives YouUse
__FILE__Current source-file namePoint a message to the relevant file
__LINE__Current source-line numberFind the exact logging call or assertion
__DATE__Compilation date stringBuild identification
__TIME__Compilation time stringBuild identification
__func__Name of the current functionShow where a log originated
__FILE__, __LINE__, __DATE__, and __TIME__ are predefined macros. __func__ is a predefined identifier introduced in C99, not a preprocessor macro. Use it inside a function.
Reusable Logging Macros
#include <stdarg.h> #include <stdio.h> static void log_message( const char *level, const char *file, int line, const char *func, const char *format, ... ) { va_list args; fprintf(stderr, "[%s] %s:%d %s(): ", level, file, line, func); va_start(args, format); vfprintf(stderr, format, args); va_end(args); fputc('\n', stderr); } #define LOG_ERROR(...) \ log_message("ERROR", __FILE__, __LINE__, __func__, __VA_ARGS__) #define LOG_WARN(...) \ log_message("WARN", __FILE__, __LINE__, __func__, __VA_ARGS__) #define LOG_INFO(...) \ log_message("INFO", __FILE__, __LINE__, __func__, __VA_ARGS__) #ifdef DEBUG #define LOG_DEBUG(...) \ log_message("DEBUG", __FILE__, __LINE__, __func__, __VA_ARGS__) #else #define LOG_DEBUG(...) ((void)0) #endif
LOG_INFO("Starting application; build: %s %s", __DATE__, __TIME__); LOG_DEBUG("index=%d, capacity=%zu", index, capacity); LOG_WARN("Retrying connection: attempt %d", attempt); LOG_ERROR("Could not open configuration file");
In release builds, the disabled LOG_DEBUG(...) macro does not evaluate its arguments. Do not put side effects such as i++, function calls that change state, or cleanup work inside debug-only logging arguments.
Assertions, errno & Error Messages
#include <assert.h> void set_value(int *ptr, int value) { assert(ptr != NULL); *ptr = value; } // Define NDEBUG before <assert.h> to disable assert.
#include <errno.h> #include <string.h> FILE *file = fopen("config.txt", "r"); if (file == NULL) { LOG_ERROR("fopen failed: %s", strerror(errno)); perror("config.txt"); }
Use assert for programmer assumptions that should never be false, not for recoverable user-input or runtime failures. Never log passwords, tokens, private data, or sensitive memory contents.

Debug Tools

Build for Debugging
// GCC / Clang-style development build gcc -std=c11 -g3 -O0 -Wall -Wextra -Wpedantic \ -Wshadow -Wconversion -Wformat=2 -DDEBUG app.c -o app // Host-machine memory and undefined-behavior checks gcc -std=c11 -g3 -O1 -fno-omit-frame-pointer \ -fsanitize=address,undefined app.c -o app_san

Advanced Debugging Checklist

Before Changing Code

1. Capture the failing input.
2. Record build flags and target.
3. Reduce the failure to a minimal case.
4. Read warnings and the first sanitizer report.

When Inspecting Logs

Include a level, source location, key state, and error code. Prefer structured, short messages that explain the event and actual values.

Avoid relying on a debug print that changes timing or memory layout to “fix” the bug. Heisenbugs often need a breakpoint, watchpoint, trace buffer, or sanitizer instead of extra output.
